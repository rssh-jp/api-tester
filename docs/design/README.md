# 設計書ディレクトリ

このディレクトリには、api-tester の各機能の **設計書** を格納します。

設計書は「**どう作るか**」を定義するドキュメントです。  
対応する仕様書（`specs/SPEC-<機能名>.md`）の要件を実現するための、コンポーネント設計・型定義・ストレージ設計・実装手順を記述します。

---

## 命名規則

```
DESIGN-<機能名>.md
```

| 部分       | 説明                                            | 例                        |
| ---------- | ----------------------------------------------- | ------------------------- |
| `DESIGN-`  | 設計書であることを示す固定プレフィックス        | —                         |
| `<機能名>` | 対応する仕様書と同じ機能識別名                  | `CategoryTree`            |
| `.md`      | Markdown 形式                                   | —                         |

### 命名例

| 機能                                   | 仕様書                              | 設計書                              |
| -------------------------------------- | ----------------------------------- | ----------------------------------- |
| カテゴリーツリー                       | `SPEC-CategoryTree.md`              | `DESIGN-CategoryTree.md`            |
| リクエスト編集パネル                   | `SPEC-RequestEditor.md`             | `DESIGN-RequestEditor.md`           |
| レスポンス表示パネル                   | `SPEC-ResponseViewer.md`            | `DESIGN-ResponseViewer.md`          |
| デフォルトヘッダー継承                 | `SPEC-DefaultHeaderInheritance.md`  | `DESIGN-DefaultHeaderInheritance.md`|
| リクエストのインポート・エクスポート   | `SPEC-ImportExport.md`              | `DESIGN-ImportExport.md`            |

## 設計書に含める内容

設計書テンプレート（[DESIGN-TEMPLATE.md](./DESIGN-TEMPLATE.md)）を雛形として使用してください。

| セクション             | 内容                                                                     |
| ---------------------- | ------------------------------------------------------------------------ |
| **対応仕様書**         | この設計書が対応する仕様書のファイル名・リンク                           |
| **コンポーネント設計** | 新規追加・変更する React コンポーネントの構成・Props・責務               |
| **型定義変更**         | TypeScript の型・インターフェース・Zod スキーマなどの追加・変更           |
| **ストレージ変更**     | localStorage / IndexedDB のキー・構造の追加・変更、マイグレーション方針  |
| **API ルート変更**     | Next.js の API Route（`app/api/`）の追加・変更                           |
| **実装手順**           | 作業を細分化したステップ（PR 分割の指針にもなる）                        |
| **テスト計画**         | ユニットテスト・統合テスト・E2E テストの方針と主なテストケース            |
| **備考**               | 技術的な決定事項・代替案の検討結果・将来の拡張に向けた注意点             |

---

## 新規設計書の作成手順

```bash
# テンプレートをコピーして新規設計書を作成
cp docs/design/DESIGN-TEMPLATE.md docs/design/DESIGN-<機能名>.md
```

コピー後、各セクションを記入してください。  
設計書を作成する前に、対応する仕様書（`docs/specs/SPEC-<機能名>.md`）が存在することを確認してください。
