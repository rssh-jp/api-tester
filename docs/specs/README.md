# 仕様書ディレクトリ

このディレクトリには、api-tester の各機能の **仕様書** を格納します。

仕様書は「**何を作るか**」を定義するドキュメントです。  
実装方法（How）には踏み込まず、機能の目的・要件・振る舞いを記述します。

---

## 命名規則

```
SPEC-<機能名>.md
```

| 部分       | 説明                                           | 例                        |
| ---------- | ---------------------------------------------- | ------------------------- |
| `SPEC-`    | 仕様書であることを示す固定プレフィックス       | —                         |
| `<機能名>` | PascalCase またはハイフン区切りの機能識別名    | `CategoryTree`            |
| `.md`      | Markdown 形式                                  | —                         |

### 命名例

| 機能                                   | ファイル名                          |
| -------------------------------------- | ----------------------------------- |
| カテゴリーツリー                       | `SPEC-CategoryTree.md`              |
| リクエスト編集パネル                   | `SPEC-RequestEditor.md`             |
| レスポンス表示パネル                   | `SPEC-ResponseViewer.md`            |
| デフォルトヘッダー継承                 | `SPEC-DefaultHeaderInheritance.md`  |
| リクエストのインポート・エクスポート   | `SPEC-ImportExport.md`              |

## 仕様書に含める内容

仕様書テンプレート（[SPEC-TEMPLATE.md](./SPEC-TEMPLATE.md)）を雛形として使用してください。

| セクション         | 内容                                                             |
| ------------------ | ---------------------------------------------------------------- |
| **背景・目的**     | なぜこの機能が必要か、解決したい課題                             |
| **機能要件**       | 機能が満たすべき条件（ユーザーが何をできるか）                   |
| **非機能要件**     | パフォーマンス・セキュリティ・アクセシビリティなどの品質要件     |
| **UI/UX 設計**     | 画面レイアウト・操作フロー・表示仕様（ワイヤーフレーム等も可）   |
| **API 設計**       | エンドポイント・リクエスト/レスポンス形式（外部 API 連携がある場合） |
| **データモデル変更** | localStorage / IndexedDB / サーバー側 DB のスキーマ変更         |
| **受け入れ条件**   | 機能完成の判断基準（Given-When-Then 形式を推奨）                 |
| **備考**           | 関連 Issue・PR・参考リンク・決定事項の記録                       |

---

## 新規仕様書の作成手順

```bash
# テンプレートをコピーして新規仕様書を作成
cp docs/specs/SPEC-TEMPLATE.md docs/specs/SPEC-<機能名>.md
```

コピー後、各セクションを記入してください。  
不要なセクションは削除して構いませんが、**背景・目的** と **受け入れ条件** は必須です。
